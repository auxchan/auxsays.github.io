release_summary: "More than 10,000 enterprises have used the GitHub Code Quality public preview to detect maintainability\
  \ and reliability issues, enforce quality gates, and track code coverage. On July 20, 2026 , Code Quality moves from public\
  \ preview to general availability as a purchasable product.\n\n\n We understand that any change is significant to our customers,\
  \ especially when it relates to billing, so we are sharing this update early to help you plan and prepare.\n\n\n What’s\
  \ changing on July 20, 2026\n\n GitHub Code Quality will become a paid product. Pricing will be $10 per active committer\
  \ per month on enabled repositories, plus usage-based consumption for AI-powered capabilities (e.g., Copilot code review,\
  \ AI-assisted detection, and Copilot Autofix). Deterministic CodeQL analysis will consume GitHub Actions minutes.\n New\
  \ capabilities will be available. Code Quality will support organization-wide deployment, organization-level quality dashboards,\
  \ code coverage enforcement via rulesets, repository- and organization-level quality scoring, and APIs for both enablement\
  \ and findings management.\n\n What you pay for\n GitHub Code Quality will be priced as a base subscription plus metered\
  \ usage. There are three components:\n\n\n\n Per-committer license, $10 per active committer per month: Covers your access\
  \ to Code Quality on enabled repositories, including findings, scoring, rulesets integration, security and quality overview\
  \ integration, organization-wide deployment, and quality gates that block pull request merges on maintainability, reliability,\
  \ or coverage thresholds.\n AI-powered work, usage-based billing: Applies when AI-powered capabilities run on code-quality–enabled\
  \ repositories. Copilot code review is the most familiar of these and consumes usage when it analyzes pull requests. AI-assisted\
  \ detection and Copilot Autofix generation also consume usage on the same model.\n Deterministic analysis, GitHub Actions\
  \ minutes: CodeQL-powered maintainability and reliability scans run on Actions. Fast, predictable, and free of AI usage\
  \ charges.\n\n Availability\n GitHub Code Quality is available on GitHub Enterprise Cloud and GitHub Team plans. Code Quality\
  \ is not available on GitHub Enterprise Server.\n\n\n If you no longer wish to use Code Quality once it becomes a paid product,\
  \ disable it on your repositories before July 20. You can disable Code Quality across repositories from your organization\
  \ settings pages starting today.\n\n\n Learn more in our docs about Code Quality .\n\n\n\n The post GitHub Code Quality\
  \ generally available July 20, 2026 appeared first on The GitHub Blog ."
