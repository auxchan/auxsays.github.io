release_summary: "Dependabot can now access internal repositories hosted in other organizations within your enterprise.\n\n\
  \n Consider the situation where you have a dependency hosted in an internal GitHub repository. This repository is in the\
  \ same enterprise as the project that uses the dependency, but it’s in a different organization. In this situation, you\
  \ can now grant Dependabot the ability to access all internal repositories, and you can do this from your enterprise’s Advanced\
  \ Security Policies page.\n\n\n Previously, Dependabot could only access repositories within the same organization, which\
  \ meant cross-organization dependencies in internal repositories couldn’t receive automatic updates. With this change, enterprise\
  \ and organization administrators can grant Dependabot access to all internal repositories across their enterprise, so Dependabot\
  \ can open pull requests for dependencies regardless of which organization hosts them.\n\n\n Get started\n For internal\
  \ and public repositories : Enable Dependabot access permanently at the enterprise level. Once enabled, all current and\
  \ future internal repositories will automatically have Dependabot access.\n\n\n This improvement is available today on github.com\
  \ and will come to enterprise server in GHES 3.22.\n\n\n To enable Dependabot for all internal repositories in your organization:\n\
  \n\n\n Navigate to your enterprise’s “Policies” page.\n Select Advanced Security from the left-side pane.\n Scroll to the\
  \ end of the page to the “Grant Dependabot access to repositories” section.\n Select the policy for repositories you’d like\
  \ to use for updates with Dependabot.\n\n Learn more\n\n Public, private, and internal repository types\n Managing Dependabot\
  \ in your enterprise\n About Dependabot security updates\n\n Have feedback about this feature? We’d love to hear from you\
  \ in GitHub Community Discussions .\n\n\n\n The post Cross-org Dependabot access for internal repositories appeared first\
  \ on The GitHub Blog ."
