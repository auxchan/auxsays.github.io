release_summary: "The Copilot usage metrics API now reports how many AI credits each user consumed per day, derived from the\
  \ same AI credits consumption data used in the usage-based billing API .\n\n\n What’s new\n A new ai_credits_used field\
  \ is now included for each user in the user-level reports and indicates the total AI credits a user consumed. This is an\
  \ overall per-user total across all of that user’s Copilot activity.\n\n\n The field is available in both the single-day\
  \ ( users-1-day ) and 28-day ( users-28-day ) user-level reports at the enterprise and organization levels.\n\n\n Why this\
  \ matters\n\n Connect consumption to value: ai_credits_used sits in the same reports you already track for usage, so you\
  \ can see consumption next to the work that drove it.\n Understand adoption across your teams: See how AI credit usage is\
  \ distributed across your enterprise or organization to recognize where Copilot is delivering the most value.\n Plan for\
  \ usage-based billing: Monitor day-over-day consumption patterns to anticipate AI credit ranges and inform budget decisions.\n\
  \n Important notes\n\n These metrics are available to enterprise administrators and organization owners who have access\
  \ to Copilot usage metrics through the REST API.\n ai_credits_used is an overall per-user total. It is not currently broken\
  \ down by feature, model, or surface.\n The field is available in the user-level reports. It is a metrics signal for analyzing\
  \ consumption, not a billed total—refer to billing for invoicing.\n\n Visit the Copilot usage metrics API documentation\
  \ to learn more. You can also join the discussion within GitHub Community .\n\n\n\n The post AI credits consumed per user\
  \ now in the Copilot usage metrics API appeared first on The GitHub Blog ."
