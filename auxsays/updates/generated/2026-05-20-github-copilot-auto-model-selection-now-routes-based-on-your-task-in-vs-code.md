  \ health metrics for a high quality, reliable, and token-efficient experience.\n\n\n How it works\n Auto weighs real-time\

  \ complexity, bug diagnosis difficulty, and tool orchestration needs to select the optimal model.\n\n\n\n Transparency :\
  \ You can see which model was used by hovering over the model response.\n Stay in control : Switch between Auto and any\
  \ specific model at any time.\n Respects your policies : Auto honors all model policies set by admins.\n Diverse model access\
  \ : Auto leverages models from multiple model families , depending on subscription type and policies. Models will change\
  \ over time.\n\n Premium request use\n Auto is billed based on the model it selects, which is currently limited to models\
  \ with 0x to 1x multipliers . Paid subscribers get a 10% discount on the model multiplier when using auto (e.g., when auto\
  \ uses a model that has a 1x multiplier, you will draw down 0.9 premium requests instead of 1).\n\n\n Getting more out of\
  \ Copilot\n Auto routes along natural cache boundaries to avoid unnecessary cache related costs. Our evaluations show gains\
  \ in token efficiencies with no quality regression, as not all tasks require a high reasoning or token-intensive model.\n\
  \n\n No setup is required. Head to VS Code and choose Auto to get started. For more information, see our documentation about\
  \ auto model selection . Join the discussion within GitHub Community .\n\n\n\n The post Auto model selection now routes\
  \ based on your task in VS Code appeared first on The GitHub Blog ."
