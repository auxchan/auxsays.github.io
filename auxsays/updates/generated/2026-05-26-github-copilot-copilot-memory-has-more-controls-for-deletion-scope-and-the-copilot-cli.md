release_summary: "Copilot Memory now includes improved memory deletion, adds a repository-level off switch, and brings further\
  \ memory controls into the Copilot CLI. Copilot Memory is in public preview and available to all paid Copilot plans.\n\n\
  \n What’s changing\n\n Deletion guidance: When you ask Copilot to forget something, it now points you to the right place\
  \ to remove the memory and down-votes the memory where voting is available.\n Repository-level off switch: Repository admins\
  \ can now disable Copilot Memory for a repository from the existing Copilot feature controls in repository settings. Repository-level\
  \ facts will no longer be stored or read. Preexisting facts are not deleted. User-level preferences are unaffected.\n /memory\
  \ command in the Copilot CLI: Use /memory on to enable Copilot Memory, /memory off to disable it, and /memory show to check\
  \ its current status. Your choice persists across sessions.\n Clearer scope at capture time: The store_memory permission\
  \ prompt now explicitly states whether the entry will be a user-level preference (i.e., visible only to you, used in your\
  \ sessions across repositories) or a repository-level fact (i.e., visible to all contributors on the repository).\n\n Managing\
  \ memories\n\n Your user-level preferences: Review or delete them in your personal Copilot Memory settings .\n Your repository’s\
  \ facts: Repository owners can review, delete, or turn off Copilot Memory for the repository under Repository Settings >\
  \ Copilot > Memory .\n\n Learn more about managing stored memories .\n\n\n Share your feedback\n For more information, see\
  \ About GitHub Copilot Memory .\n\n\n We’re continuing to evolve Copilot Memory and plan to bring it to more features in\
  \ the future. To share feedback or join the discussion, visit the GitHub Community .\n\n\n\n The post Copilot Memory has\
  \ more controls for deletion, scope, and the Copilot CLI appeared first on The GitHub Blog ."
