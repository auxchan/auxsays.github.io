release_summary: "This changelog covers VS Code v1.123 through v1.127 , shipped throughout June and early July 2026.\n\n\n\
  \ The latest VS Code releases build on the Copilot experience developers use every day, making it easier to manage agent\
  \ work, understand usage, choose the right models, and stay in the flow. From a smarter integrated browser to parallel sessions,\
  \ clearer cost visibility, Marketplace model discovery, and sharper Autopilot behavior, these updates help developers move\
  \ faster while staying in control.\n\n\n Integrated browser updates\n The integrated browser brings more of your web workflow\
  \ into VS Code, including search, screenshots, remote browsing, and agent-driven validation.\n\n\n\n Agentic browser tools\
  \ are now generally available: This is enabled by default, allowing agents to navigate pages, inspect content, capture screenshots,\
  \ and validate web apps directly in VS Code. For more information, see our changelog for the preview release of agentic\
  \ browser tools .\n Improved browsing experience: The integrated browser now has favorites, history, and web search, as\
  \ well as permissions for camera, location, microphone, and more.\n New ways to give web page context to the agent: You\
  \ now have options for adding a screenshot or area screenshot to chat.\n Browse from remote workspaces : When the integrated\
  \ browser is opened in a remote workspace, HTTP(S) web traffic can now be proxied through the remote connection. This feature\
  \ is in public preview.\n\n\n\n\n Parallel sessions and chats\n The Agents window continues to grow with enhancements for\
  \ keeping parallel sessions and chats organized and easy to navigate.\n\n\n\n Run sessions side by side: Keep separate agent\
  \ tasks visible and active at the same time.\n\n\n Use multiple chats in one session: Break a larger change into focused\
  \ workstreams, so you can keep implementation, review, testing, and docs separate while still managing the overall task\
  \ in one place.\n\n\n\n\n Agent session organization: Tidy up your Agents window by grouping related sessions or dragging\
  \ and dropping your sessions to rearrange them.\n\n\n\n\n\n\n\n\n\n Cost visibility and improvements\n Cost visibility is\
  \ clearer across whole sessions, delegated work, and additional Copilot spend.\n\n\n\n See total session cost: Understand\
  \ credit usage across the full chat, not just one request.\n\n\n Track additional usage: Review usage from the GitHub Copilot\
  \ status dashboard.\n\n\n\n\n Inspect subagent usage: See credit usage for individual subagent sections when work is delegated.\n\
  \n\n\n\n\n\n\n Model providers from the Marketplace\n Model provider discovery now starts directly from VS Code, making\
  \ it easier to add and tune models for different tasks.\n\n\n\n Discover providers from one place: Use the Language Models\
  \ editor to find model provider extensions.\n\n\n Install from Marketplace: Open filtered Marketplace results for extensions\
  \ that contribute models.\n\n\n\n\n Customize models faster: Adjust context size and reasoning effort from a unified picker.\n\
  \n\n\n\n\n\n\n\n\n Autopilot improvements\n Autopilot, the permission level that lets agents act without checking in at\
  \ every step, is now more hands-off and better at seeing tasks through.\n\n\n\n Better task completion: Agents are smarter\
  \ about determining when the requested work is actually finished.\n\n\n More independent progress: Agents can continue working\
  \ through steps with less manual steering.\n\n\n\n\n\n\n\n\n\n More agent workflow wins\n\n\n Session sync and chronicle:\
  \ Sync chat sessions to your GitHub account and search coding history across machines and workspaces.\n\n\n\n\n Gutter feedback:\
  \ Leave comments on an agent’s changes directly from the editor gutter.\n\n\n\n\n Smarter pull request creation: Create\
  \ a pull request from a session with the title and description generated from session context.\n\n\n\n\n 1M context windows:\
  \ Work with compatible Anthropic and OpenAI models using much larger context windows for bigger codebases and longer conversations.\n\
  \n\n\n\n Model hover cards: See a quick model descriptor and jump directly to relevant configuration options.\n\n\n\n\n\
  \ Official Ollama extension: Move from the built-in Ollama provider to the official VS Code extension for faster model and\
  \ capability updates.\n\n\n\n\n Managed Copilot settings: Deliver Copilot configuration through native device management\
  \ on Windows and macOS.\n\n\n\n\n File-based managed settings: Apply managed Copilot settings from a JSON file for machines\
  \ that are not enrolled in device management.\n\n\n\n\n MCP OAuth credentials: Configure preregistered OAuth client IDs\
  \ and keep client secrets in VS Code secret storage.\n\n\n\n\n Extension auto-update delay: A two-hour delay is applied\
  \ before automatically installing newly published extension versions, giving teams a safety buffer.\n\n\n\n\n Workspace\
  \ Trust improvements: Browse new folders safely first, then trust the folder when ready.\n\n\n\n\n Taken together, these\
  \ updates continue to evolve VS Code for agentic development. Agents can work across longer-running tasks, developers get\
  \ clearer visibility into costs and models, and teams gain better tools for building and shipping with confidence.\n\n\n\
  \ For the full details, browse the complete release notes across these versions: Visual Studio Code 1.123, 1.124, 1.125,\
  \ 1.126 and & 1.127 .\n\n\n Install the latest version of Visual Studio Code today, and as always happy coding!\n\n\n\n\
  \ The post GitHub Copilot in Visual Studio Code, June 2026 releases appeared first on The GitHub Blog ."
