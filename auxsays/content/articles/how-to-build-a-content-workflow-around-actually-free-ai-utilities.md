# Start with the bottleneck, not the app

A usable workflow begins with the part that hurts the most.

For some people that is ideation. For others it is cleanup, repurposing, metadata, or export formatting. The point is not to add five AI tools because you can. The point is to remove one meaningful point of drag.

## A simple stack mindset

1. Use one tool for idea generation or scripting
2. Use one tool for asset cleanup or formatting
3. Keep the final assembly in a tool you trust

## Avoid these mistakes

- Building a stack you cannot explain a month later
- Letting the cheapest tool control your final output
- Depending on hidden quotas for a client-facing workflow
