# AUXSAYS Patch Record Typo Repair

This repair addresses the ComfyUI generated patch pages where GitHub's autogenerated heading `What's Changed` rendered awkwardly as `What’ s Changed` in the official notes section.

## Cause
The affected records came from GitHub Releases. GitHub's autogenerated release heading was captured directly into `official_patch_notes_body` as Markdown:

```md
## What's Changed
```

On the AUXSAYS update page typography, that heading rendered awkwardly. The problem was isolated to generated GitHub release records that retained the GitHub heading verbatim.

## Fix
- Existing generated ComfyUI records were normalized from `What's Changed` to `Changes`.
- Existing ComfyUI summaries were regenerated so they no longer start with raw Markdown like `## What's Changed * ...`.
- Future generated records normalize common GitHub release headings before writing front matter.
- Official patch-note heading CSS was hardened to use normal Inter typography instead of display-style spacing.

## Files changed
- `auxsays/scripts/lib/normalize.py`
- `auxsays/scripts/lib/write_update_record.py`
- `auxsays/assets/css/auxsays-custom.css`
- `auxsays/updates/generated/2026-04-17-comfyui-0-19-3.md`
- `auxsays/updates/generated/2026-04-27-comfyui-0-20-1.md`
